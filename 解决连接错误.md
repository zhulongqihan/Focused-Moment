# 解决 "ERR_CONNECTION_REFUSED" 错误

## 问题描述
打开 exe 文件后显示：
```
嗯… 无法访问此页面
localhost 拒绝连接。
ERR_CONNECTION_REFUSED
```

## 原因分析
这个错误说明应用在尝试连接 localhost，而不是使用打包的前端资源。可能的原因：

1. **运行了错误的 exe 文件**
   - 可能运行的是旧的 `tauri-app.exe` 而不是新的 `focused-moment.exe`
   
2. **前端资源没有正确打包**
   - 构建时没有先运行 `npm run build`
   - build 目录不存在或内容不完整

3. **exe 文件是旧版本**
   - 文件被缓存或没有正确更新

## 解决方案

### 方案1：使用完全重建脚本（推荐）

1. **关闭所有 Focused Moment 实例**
   - 确保没有任何 exe 正在运行
   - 检查任务管理器，结束所有相关进程

2. **运行完全重建脚本**
   ```
   双击 "完全重建.bat"
   ```
   
3. **等待构建完成**
   - 脚本会自动清理旧文件
   - 重新构建前端和后端
   - 复制新的 exe 到根目录

4. **运行新的 exe**
   ```
   双击 "Focused-Moment.exe"
   ```

### 方案2：手动重建

1. **关闭所有实例**
   ```bash
   # 在任务管理器中结束所有 focused-moment.exe 和 tauri-app.exe 进程
   ```

2. **删除旧文件**
   ```bash
   # 删除这些文件（如果存在）
   del src-tauri\target\release\tauri-app.exe
   del src-tauri\target\release\focused-moment.exe
   del Focused-Moment.exe
   ```

3. **清理 build 目录**
   ```bash
   rmdir /s /q build
   ```

4. **重新构建前端**
   ```bash
   npm run build
   ```

5. **重新构建后端**
   ```bash
   cd src-tauri
   cargo build --release
   cd ..
   ```

6. **复制新的 exe**
   ```bash
   copy src-tauri\target\release\focused-moment.exe Focused-Moment.exe
   ```

7. **运行应用**
   ```bash
   # 双击 Focused-Moment.exe
   ```

### 方案3：使用 Tauri CLI（最可靠）

1. **关闭所有实例**

2. **使用 Tauri 构建命令**
   ```bash
   npm run tauri:build
   ```
   
   这个命令会：
   - 自动运行 `npm run build`
   - 自动构建 Rust 后端
   - 正确打包所有资源

3. **运行生成的 exe**
   ```bash
   # 位置：src-tauri\target\release\focused-moment.exe
   ```

## 验证步骤

### 1. 检查 build 目录
```bash
dir build
```
应该看到：
- index.html
- _app/ 目录
- favicon.png
- 其他静态资源

### 2. 检查 exe 文件大小
```bash
dir Focused-Moment.exe
```
应该约为 10-11 MB

如果文件很小（< 5 MB），说明前端资源没有打包进去。

### 3. 检查文件修改时间
确保 exe 文件的修改时间是最新的（刚刚构建的）。

### 4. 运行诊断脚本
```bash
双击 "诊断问题.bat"
```
查看所有检查项是否通过。

## 常见错误

### 错误1：文件被占用
```
文件正由另一进程使用
```
**解决**：关闭所有 Focused Moment 实例，包括任务管理器中的后台进程。

### 错误2：build 目录不存在
```
找不到 build 目录
```
**解决**：运行 `npm run build`

### 错误3：cargo 命令找不到
```
'cargo' 不是内部或外部命令
```
**解决**：
1. 确保已安装 Rust
2. 重启命令行窗口
3. 或使用完全重建脚本（已配置环境变量）

## 预防措施

### 以后构建时
1. **始终使用构建脚本**
   - 使用 "构建应用.bat" 或 "完全重建.bat"
   - 不要直接运行 `cargo build`

2. **确保前端先构建**
   - 先运行 `npm run build`
   - 再运行 `cargo build --release`

3. **使用正确的 exe**
   - 运行 `Focused-Moment.exe`（根目录）
   - 或 `focused-moment.exe`（src-tauri/target/release/）
   - 不要运行 `tauri-app.exe`（旧文件）

## 技术细节

### Tauri 如何加载前端
1. **开发模式**：连接到 `http://localhost:1420`（Vite 开发服务器）
2. **生产模式**：从 `frontendDist` 目录加载静态文件

### 配置文件
- `src-tauri/tauri.conf.json`：
  ```json
  "build": {
    "frontendDist": "../build"
  }
  ```

### 构建流程
1. `npm run build` → 生成 `build/` 目录
2. `cargo build --release` → 将 `build/` 打包进 exe
3. exe 运行时从内部加载前端资源

## 仍然无法解决？

如果尝试了所有方案仍然无法解决：

1. **检查防火墙**
   - 某些防火墙可能阻止应用加载本地资源

2. **检查杀毒软件**
   - 某些杀毒软件可能干扰应用运行

3. **尝试开发模式**
   ```bash
   npm run tauri:dev
   ```
   如果开发模式正常，说明是构建配置问题。

4. **查看日志**
   - 按 F12 打开开发者工具
   - 查看 Console 和 Network 标签
   - 截图错误信息

5. **提交 Issue**
   - 在 GitHub 上提交问题
   - 附上错误截图和日志
