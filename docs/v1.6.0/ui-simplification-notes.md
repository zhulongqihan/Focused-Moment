# v1.6.0 界面减法说明

这轮调整的目标不是删功能，而是把不该在主界面里长期占位的解释性内容收回文档。

## 从界面里收走了什么

- 导航区域的说明性摘要
- 专注页、待办页、复盘页里偏引导性的整段说明
- 原 `开发者信息` 页面中的模块路线、运行机制和内部实现描述

## 仍然保留在界面的内容

- 用户会直接操作的按钮、筛选器、模式切换和状态值
- 备份导出与恢复
- 显示模式切换
- 当前版本与本地数据相关信息

## 为什么这样做

- 主界面首先应该服务“马上开始使用”
- 说明文字越多，越容易把任务、计时、趋势这些真正重要的内容挤到后面
- 技术背景和版本说明更适合在仓库文档、release note 和 README 中长期维护
