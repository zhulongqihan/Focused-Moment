# Task 3.1 实施总结：单抽逻辑实现

## 任务概述

实现了明日方舟抽卡系统的单次抽卡功能，包括完整的货币验证、概率计算、干员选择、保底计数器更新和数据持久化。

## 实现的功能

### 1. 核心功能

#### `perform_single_gacha` 函数
- **位置**: `src-tauri/src/gacha/single_pull.rs`
- **功能**: 执行单次抽卡的完整流程

#### 实现的需求验证

✅ **需求 3.1**: 验证合成玉余额（>= 600）
- 从数据库加载当前货币余额
- 检查合成玉是否足够

✅ **需求 3.2**: 合成玉不足时返回错误
- 返回清晰的错误信息，包含所需数量和当前数量
- 示例: "合成玉不足，需要 600 合成玉，当前仅有 500 合成玉"

✅ **需求 3.3**: 调用概率计算函数确定稀有度
- 集成 `calculate_gacha_rarity` 函数
- 根据保底计数器计算稀有度

✅ **需求 3.4**: 从干员池中随机选择干员
- 实现了 40 个干员的干员池（每个稀有度 10 个）
- 使用 `rand` 库进行随机选择

✅ **需求 3.5**: 将干员添加到用户收藏
- 调用 `save_operator` 保存到数据库
- 生成唯一的干员 ID

✅ **需求 3.6**: 更新保底计数器
- 获得 6★ 时重置为 0
- 未获得 6★ 时加 1

✅ **需求 3.7**: 保存抽卡历史记录
- 记录完整的抽卡信息
- 包含时间戳、类型、干员、消耗货币、保底计数器变化

### 2. 干员池实现

实现了 4 个稀有度共 40 个干员：

**6★ 干员（10 个）**:
- 银灰、艾雅法拉、能天使、闪灵、塞雷娅
- 推进之王、陈、伊芙利特、麦哲伦、莫斯提马

**5★ 干员（10 个）**:
- 德克萨斯、幽灵鲨、蓝毒、白面鸮、赫默
- 临光、雷蛇、天火、初雪、红

**4★ 干员（10 个）**:
- 夜刀、杰西卡、调香师、末药、古米
- 白雪、远山、梅、砾、安塞尔

**3★ 干员（10 个）**:
- 芬、香草、翎羽、玫兰莎、卡缇
- 米格鲁、克洛丝、安德切尔、史都华德、阿消

### 3. 数据流程

```
1. 加载货币余额
2. 验证合成玉 >= 600
3. 加载保底计数器
4. 计算稀有度（调用概率引擎）
5. 从干员池随机选择干员
6. 更新保底计数器
7. 扣除 600 合成玉
8. 保存干员到数据库
9. 更新货币余额
10. 更新保底计数器
11. 保存抽卡历史记录
12. 返回抽卡结果
```
