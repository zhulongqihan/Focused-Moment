# Focused Moment

Focused Moment 是一款面向 Windows 的轻量效率工具，把专注计时、待办清单和数据复盘放进同一个安静、克制的桌面界面里。

它适合这样的使用方式：
- 开始一轮正向计时或番茄钟
- 把这轮专注和一个待办关联起来
- 在结束后自动留下记录
- 回到数据页看看自己最近的节奏

## 你可以用它做什么

- `专注计时`
  支持正向计时和番茄钟，两种方式都可以记录正在做的事情。

  <p>
    <img src="./专注计时.png" alt="专注计时主题插画" width="260" />
  </p>
  <p><sub>对应专注计时与番茄钟节奏的内容插画。</sub></p>
- `待办管理`
  可以添加任务、设置日期、可选的开始时间和重要程度，并在专注时直接关联；也可以按关键字搜索、按状态筛选、按时间或重要程度排序。

  <p>
    <img src="./管理待办.png" alt="管理待办主题插画" width="260" />
  </p>
  <p><sub>对应任务整理与待办安排的内容插画。</sub></p>
- `数据复盘`
  可以查看累计时长、专注次数、最近趋势和按日节奏，也可以按时间范围筛选、删除测试记录或清理当前范围；趋势图支持更直观的层次展示，并可通过点位查看某一天的详细信息。

  <p>
    <img src="./状态复盘.png" alt="状态复盘主题插画" width="260" />
  </p>
  <p><sub>对应复盘、趋势观察与状态回看的内容插画。</sub></p>
- `异常恢复`
  如果应用在专注过程中意外关闭，再次打开时会优先恢复上一轮尚未结束的计时状态，尽量减少中断带来的丢失。
- `本地快照保护`
  保存任务、专注记录和运行中状态前，会先保留上一份本地快照，给后续回退和排查留一层保险。

  <p>
    <img src="./本地保护.png" alt="本地保护主题插画" width="260" />
  </p>
  <p><sub>对应本地保存、快照保护与数据安全的内容插画。</sub></p>
- `计时设置与提醒`
  可以单独设置番茄专注时长、番茄休息时长和正向计时提醒目标；到点后支持系统通知和窗口/任务栏提醒。
- `更稳的启动体验`
  现在会先把计时和待办主界面拉起来，再在后台补齐复盘记录与统计；即使历史数据加载较慢，也尽量不影响先开始使用。
- `更轻的主界面`
  主界面默认只保留当前专注真正要用的信息；最近专注记录改成按需展开，避免一打开就让整页信息挤在一起。
- `更清晰的信息分层`
  默认主界面只放“现在马上要操作的内容”；托盘说明、轮次、恢复提示和最近记录被收进单独的专注详情区，需要时再展开看。
- `更稳的渲染保护`
  默认优先使用更轻的视觉模式，减少多层模糊、毛玻璃和阴影对 Windows 桌面环境的压力；如果你想保留更强的视觉效果，也可以在开发者页切回标准模式。
- `更完整的 Windows 外观`
  当前测试版 exe 也会使用图形子系统启动，不再额外弹出黑色控制台窗口；应用图标也同步换成了新的正式临时版。
- `更克制的列表密度`
  主界面的最近专注记录默认只保留近一周窗口；任务页会把已完成任务收进可折叠分区，减少长期使用后的拥挤感。
- `后台常驻`
  点击右上角 `隐藏` 后，应用会驻留在系统托盘里继续运行；点击 `退出` 才会完全关闭。
- `单例运行`
  同一时间只保留一个应用实例；如果你再次双击程序，会直接回到已经打开的那一份。
- `专注驾驶舱`
  当前主界面已经收敛为左侧轻量导航与专注驾驶舱布局，计时器、当前事务和核心操作会更集中。
- `科技感鼠标粒子`
  鼠标移动时会出现轻量的冷青色线性粒子和脉冲效果，只作为氛围点缀，不遮挡点击和输入。
- `动态专注仪表盘`
  计时区现在会用轻量进度环展示番茄钟、休息或正向提醒目标；没有设置正向提醒时，也会保持自由计时状态展示。

## 日常使用方式

### 1. 开始专注
- 打开 `开始专注`
- 输入这次要做的事情
- 选择 `正向计时` 或 `番茄钟`
- 在 `计时模式` 下方查看当前专注、休息和正向提醒时长；如果需要，可以展开 `计时设置` 调整番茄时长、休息时长或正向提醒目标
- 点击 `开始计时`

### 2. 管理今天的待办
- 打开 `管理待办`
- 添加今天要做的任务
- 给任务设置日期、可选的开始时间和重要程度
- 在专注页面里可以直接把一轮专注关联到某个任务

### 3. 查看最近状态
- 打开 `查看复盘`
- 看总专注时长、最近趋势和每天的沉淀节奏
- 可以切换 `今天 / 近 7 天 / 近 30 天 / 全部 / 自定义`
- 如果只是测试数据，可以删除单条记录、清理当前范围，或者直接清空本地数据重新开始

### 4. 查看内部信息
- 打开 `开发者信息`
- 这里会集中显示版本号、当前状态、运行方式和当前纯效率工具主线的内部说明
- 正常日常使用时可以完全忽略这一页

## 数据与隐私

- 当前所有任务、专注记录和统计数据都默认保存在本地
- 不依赖在线账号
- 不需要联网才能完成日常使用

## Windows 使用方式

如果你已经拿到构建好的程序，根目录里通常会看到这些文件：
- `Focused Moment.exe`
- `Focused Moment vX.Y.Z.exe`
- `Focused Moment Setup.exe`
- `Focused Moment Setup vX.Y.Z.exe`

推荐优先使用带版本号的文件进行测试或留档。

## 如果你要自己构建

启动开发环境：

```bash
pnpm tauri dev
```

导出调试版：

```bash
pnpm package:debug
```

导出正式版：

```bash
pnpm package:release
```

把已经构建好的当前版本同步到 GitHub Release：

```bash
pnpm release:github
```

如果你希望按固定顺序一口气完成：
`构建 -> 推送 main -> 打 tag -> 推送 tag -> GitHub Release`

```bash
pnpm release:ship
```

只清理根目录旧构建产物：

```bash
pnpm clean:artifacts
```
